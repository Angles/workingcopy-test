General observations - regex in textastic

Regex [ ] ( ) { } are NOT escaped! You are lucky.

For multi-line search use [\s\S]* for the wildcard.

Some REGEX EXAMPLES

Note in the next few examples we look for actual [ brackets. Keep in mind:

  1. any escaped brackets below mean actual text brackets.
  2. do NOT confuse these with regex char set grouping brackets, they are NOT escaped in textastic.
  3. for REGEX in TEXTASTIC the regex [ ] are not escaped.
  4. related example: use [\s\S]* for multi-line search.

SCENARIO 1

find a starting script tag that is not at the start of a line. Make it the start of a line with an extra empty line above it.

example find this text:
</ul><script type="text/javascript">

SEARCH

([^\n])(<script)

REPLACE

$1\n\n$2

Example result text after replace action:
</ul>

<script type="text/javascript">

SCENARIO 2

This is for if script tag is at start of line but there is not an empty line above it.

SEARCH

([^\n])(\n)(<script)

REPLACE

$1$2$2$3

place a line break after a closing script tag.
look for closing script tag followed by a non-line break, example find text,

SEARCH

(</script>)([^\n])

REPLACE

</div>
